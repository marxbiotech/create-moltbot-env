# Migration: 0.4.0 → 0.5.0

## Summary

This migration moves the WebSocket relay into `@marxbiotech/create-moltbot-env` as a `node` subcommand and replaces the hardcoded `/acp` bypass route with a per-environment random `NODE_BYPASS_ROUTE`. This makes the bypass route unpredictable for improved security.

**Prerequisites:** Deploy moltbot-app with dynamic bypass route support (NODE_BYPASS_ROUTE env var) before applying these changes.

## Steps

### 1. Add `NODE_BYPASS_ROUTE` to each overlay

For each overlay in `overlays/*/wrangler.jsonc`, add a `NODE_BYPASS_ROUTE` var with a random hex path. Generate a unique value per environment:

```bash
# Generate a random route for each overlay
for dir in overlays/*/; do
  ROUTE="/node-$(openssl rand -hex 8)"
  echo "$dir → $ROUTE"
done
```

Add the var to the `vars` block in each overlay's `wrangler.jsonc`:
```jsonc
"NODE_BYPASS_ROUTE": "/node-<generated-hex>",
```

### 2. Update `scripts/sync-access.sh`

Replace the hardcoded `acp|/acp|MOLTBOT_GATEWAY_TOKEN` bypass config with a dynamic read from overlay vars.

**Replace** the `BYPASS_CONFIGS` array section with:

```bash
# Bypass apps: name_suffix|path|secret_name
# Node bypass route is read dynamically from overlay vars (per-environment random path).
NODE_BYPASS_ROUTE=$($STRIP "$OVERLAY_DIR/wrangler.jsonc" | jq -r '.vars.NODE_BYPASS_ROUTE // empty')
BYPASS_CONFIGS=(
  "telegram-webhook|/telegram/webhook|TELEGRAM_WEBHOOK_SECRET"
  "slack-events|/slack/events|SLACK_SIGNING_SECRET"
)
if [[ -n "$NODE_BYPASS_ROUTE" ]]; then
  BYPASS_CONFIGS+=("node-bypass|${NODE_BYPASS_ROUTE}|MOLTBOT_GATEWAY_TOKEN")
fi
```

### 3. Update `scripts/create-env.sh`

In the wrangler.jsonc generation section (inside the `{ echo ... } > "$OVERLAY_DIR/wrangler.jsonc"` block), add a `NODE_BYPASS_ROUTE` line before the `WORKER_URL` line:

```bash
  echo "    \"NODE_BYPASS_ROUTE\": \"/node-$(openssl rand -hex 8)\","
```

### 4. Update `Makefile`

Add `node` to the `.PHONY` line and add the `node` target:

```makefile
# Start paired node: relay + openclaw node via create-moltbot-env CLI
# Override node name: make node NODE_NAME=my-custom-name
NODE_NAME ?= $(ENV_NAME)-$(shell hostname -s)
node:
	@npx @marxbiotech/create-moltbot-env node --env $(ENV_NAME) --node-name "$(NODE_NAME)"
```

If you have existing `acp-setup`, `acp`, or `node` targets that reference `ws-relay.mjs` or hardcoded `/acp`, remove or replace them.

### 5. Delete removed files

Remove the following files that are no longer needed (relay moved to create-moltbot-env):

- `scripts/ws-relay.mjs`
- `package.json` (ws dependency no longer needed at repo root)
- `package-lock.json`

### 6. Run sync-access for each overlay

```bash
for dir in overlays/*/; do
  env=$(basename "$dir")
  (cd "$dir" && make sync-access)
done
```

This will create the new bypass Access app with the random route path and remove the old `/acp` bypass app.

### 7. Deploy all overlays

Deploy each overlay to activate the `NODE_BYPASS_ROUTE` env var in the Worker.

### 8. Bump version

Update `.moltbot-env.json` meta version to `0.5.0`.

## Verification

1. `cd overlays/<env> && make node` — node connects via the new relay
2. Old `/acp` path returns 404 (no route registered)
3. `make sync-access` shows the new `node-bypass` app with the random path
