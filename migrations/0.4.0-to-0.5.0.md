# Migration: 0.4.0 → 0.5.0

## Summary

This migration replaces the hardcoded `/acp` bypass route with a per-environment `NODE_ROUTE` protected by CF Access Service Token authentication. The WebSocket relay moves from `scripts/ws-relay.mjs` into `@marxbiotech/create-moltbot-env` as a `node` subcommand with reconnect, heartbeat, and graceful shutdown.

Additionally, `deploy.sh` gains two enhancements:
- **Skills injection** — copies env-managed skills from `skills/` into the app repo at deploy time, controlled by per-overlay `skills.txt` files
- **Secret type coercion** — auto-converts non-string values in `secrets.json` to JSON strings for `wrangler secret bulk` compatibility

**Prerequisites:**
- Deploy moltbot-app with `NODE_ROUTE` support (the node route middleware must be present)
- Create a CF Access Service Token in Cloudflare Zero Trust dashboard
- Store the Service Token credentials in macOS Keychain

## Steps

### 1. Create CF Access Service Token

Go to **Cloudflare Zero Trust → Settings → Service Auth → Create Service Token**.

Save the credentials to macOS Keychain:

```bash
security add-generic-password -a "cf-node-service-token" -s "cf-node-client-id" -w "<Client-ID>" -U
security add-generic-password -a "cf-node-service-token" -s "cf-node-client-secret" -w "<Client-Secret>" -U
```

Note the Service Token UUID from the dashboard (needed for step 2).

### 2. Add `NODE_ROUTE` and `NODE_SERVICE_TOKEN_ID` to each overlay

For each overlay in `overlays/*/wrangler.jsonc`, add two vars:

```bash
# Generate a random route for each overlay
for dir in overlays/*/; do
  ROUTE="/node-$(openssl rand -hex 8)"
  echo "$dir → $ROUTE"
done
```

Add to the `vars` block:
```jsonc
"NODE_ROUTE": "/node-<generated-hex>",
"NODE_SERVICE_TOKEN_ID": "<service-token-uuid>",
```

`NODE_SERVICE_TOKEN_ID` restricts the Access app to only accept the specified Service Token. If omitted, any valid service token for the account is accepted.

### 3. Replace `scripts/sync-access.sh`

Replace the entire file with the version from the template. Key changes:
- `BYPASS_CONFIGS` renamed to `ACCESS_CONFIGS` with a 4th field: `policy_type`
- Reads `NODE_ROUTE` and `NODE_SERVICE_TOKEN_ID` from overlay vars
- Node route uses `service_auth` policy type (`non_identity` decision) instead of `bypass`
- Webhook routes unchanged (`bypass` policy)

The new `ACCESS_CONFIGS` section:
```bash
# Access apps: name_suffix|path|secret_name|policy_type
# policy_type: "bypass" (default) or "service_auth"
OVERLAY_JSON=$($STRIP "$OVERLAY_DIR/wrangler.jsonc")
NODE_ROUTE=$(echo "$OVERLAY_JSON" | jq -r '.vars.NODE_ROUTE // empty')
NODE_SERVICE_TOKEN_ID=$(echo "$OVERLAY_JSON" | jq -r '.vars.NODE_SERVICE_TOKEN_ID // empty')
ACCESS_CONFIGS=(
  "telegram-webhook|/telegram/webhook|TELEGRAM_WEBHOOK_SECRET|bypass"
  "slack-events|/slack/events|SLACK_SIGNING_SECRET|bypass"
)
if [[ -n "$NODE_ROUTE" ]]; then
  ACCESS_CONFIGS+=("node|${NODE_ROUTE}|MOLTBOT_GATEWAY_TOKEN|service_auth")
fi
```

### 4. Update `scripts/create-env.sh`

In the wrangler.jsonc generation block, add a `NODE_ROUTE` line before `WORKER_URL`:

```bash
  echo "    \"NODE_ROUTE\": \"/node-$(openssl rand -hex 8)\","
```

### 5. Update `Makefile`

Remove `acp`, `acp-setup` targets and their `.PHONY` entries (no longer functional — `openclaw acp` cannot pass Service Token headers).

Add `node` to `.PHONY` and add the `node` target:

```makefile
# Start paired node: relay + openclaw node via create-moltbot-env CLI
# Loads CF Access Service Token from Keychain for authentication.
# Override node name: make node NODE_NAME=my-custom-name
NODE_NAME ?= $(ENV_NAME)-$(shell hostname -s)
node:
	@CF_ACCESS_CLIENT_ID=$$(security find-generic-password -a "cf-node-service-token" -s "cf-node-client-id" -w 2>/dev/null) \
	CF_ACCESS_CLIENT_SECRET=$$(security find-generic-password -a "cf-node-service-token" -s "cf-node-client-secret" -w 2>/dev/null) \
	npx @marxbiotech/create-moltbot-env node --env $(ENV_NAME) --node-name "$(NODE_NAME)"
```

### 6. Delete removed files

```bash
rm -f scripts/ws-relay.mjs package.json package-lock.json
```

### 7. Add skills injection to `scripts/deploy.sh`

After step 1 (clone + checkout) and before step 2 (`npm ci`), add:

```bash
# 1b. Install env-managed skills into app repo before build
SKILLS_FILE="$OVERLAY_DIR/skills.txt"
if [[ -f "$SKILLS_FILE" ]]; then
  skill_count=0
  while IFS= read -r skill_name || [[ -n "$skill_name" ]]; do
    skill_name="${skill_name%%#*}"          # strip inline comments
    skill_name="$(echo "$skill_name" | xargs)" # trim whitespace
    [[ -z "$skill_name" ]] && continue
    skill_src="$ROOT_DIR/skills/$skill_name"
    if [[ ! -d "$skill_src" ]]; then
      echo "Error: skill '$skill_name' not found in skills/" >&2; exit 1
    fi
    mkdir -p "$WORK_DIR/skills/$skill_name"
    cp -r "$skill_src"/* "$WORK_DIR/skills/$skill_name/"
    ((skill_count++))
  done < "$SKILLS_FILE"
  [[ $skill_count -gt 0 ]] && echo "  Skills injected: $skill_count"
fi
```

This is a no-op if `skills.txt` doesn't exist — no changes needed for existing overlays.

### 8. Add secret type coercion to `scripts/deploy.sh`

In the secrets push section (step 5), replace:

```bash
sops decrypt "$OVERLAY_DIR/secrets.json" > "$secrets"
```

With:

```bash
sops decrypt "$OVERLAY_DIR/secrets.json" \
  | jq 'to_entries | map(if .value | type != "string" then .value = (.value | tojson) else . end) | from_entries' \
  > "$secrets"
```

`wrangler secret bulk` only accepts string values. This jq pipe auto-converts non-string values (e.g., `"MAX_RETRIES": 3` or `"FLAGS": {"beta": true}`) to JSON strings. Existing all-string `secrets.json` files are unaffected.

### 9. Run sync-access for each overlay

```bash
for dir in overlays/*/; do
  env=$(basename "$dir")
  (cd "$dir" && make sync-access)
done
```

This will:
- Delete old `/acp` bypass apps (if any)
- Create new `*-node` Access apps with `service_auth` policy for the `NODE_ROUTE` path
- Automatically add `NODE_ACCESS_AUD` to each overlay's `wrangler.jsonc` (enables Worker-level JWT verification)
